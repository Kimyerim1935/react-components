## Convention

### Git

<code>feat/[주제명]</code>으로 브랜치 생성

```
GIT COMMIT

feat : …
mod : …
fix : …
chore : …
doc : …
```

### PR 규칙

0. 문서는 feat/ 폴더에 작성한다.
1. main branch로 PR을 보낸다.
2. PR은 다른 참여자의 승인을 받아야 한다.
3. PR merge는 월요일마다 진행한다.

```
PR 메시지

[깃허브 아이디] 컴포넌트명 추가

```
